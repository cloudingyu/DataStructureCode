# 排序

## 题目描述

本题只允许使用C语言提交。

我们定义一个随机数生成器 `xorshift()` 函数。

```c
unsigned int xorshift(unsigned int x) {
    x ^= x << 13;
    x ^= x >> 17;
    x ^= x << 5;
    return x;
}
```

现在，告诉你两个整数 $n, x_0$，需要用这个随机数生成器生成 $x_1, x_2 \cdots x_n$。

其中，$x_i = \text{xorshift}(x_{i-1})$，$1 \le i \le n$。

然后，你需要将 $x_1, x_2 \cdots x_n$ 从小到大排序，使得 $x_1 \le x_2 \le \cdots x_n$。

对于排序后的序列，并不要求你输出整个数列。为了防止输出量太大，你只需要输出 $\sum_{i=1}^{n} i \cdot x_i$ 对 $2^{32}$ 取模的结果即可。

## 输入格式

第一行，两个整数 $n, x_0$。

## 输出格式

输出一个整数，表示答案（注意数据范围）。

## 样例 #1

### 样例输入 #1

```
3 1
```

### 样例输出 #1

```
3782878834
```

## 样例 #2

### 样例输入 #2

```
3 2147483647
```

### 样例输出 #2

```
262620378
```

## 数据范围与提示

对于 100% 的测试数据满足 $n \le 3 \times 10^7$。

一般来说，Xorshift 算法可以产生较为均匀的分布。

**提示：**

相同复杂度的算法，使用 STL 实现的时空间常数通常高于自己的手动实现。

如果你认为复杂度正确，但是超时，可以考虑把基于 STL 容器的实现改为手动实现。

注意数据范围，此题的数值属于 `unsigned int` 范畴。
